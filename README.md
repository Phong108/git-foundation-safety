**project được thực hiện như sau:**

1.Em tạo repository trên GitHub, xây dựng các chức năng cơ bản như thêm, tìm kiếm, cập nhật và xóa sinh viên.... Trong quá trình làm việc em thực hiện nhiều commit với nội dung rõ ràng và có ý nghĩa  
2.Em tạo 3 branch gồm feature-add, feature-search và feature-ui. Sau khi hoàn thành từng chức năng, em tạo Pull Request để hợp nhất code vào branch main, qua đó thực hành quy trình làm việc nhóm trên GitHub.  
3.Em thực hành tạo xung đột (merge conflict) giữa các branch và tiến hành giải quyết conflict thành công. Ngoài ra, em cũng thực hiện git revert để hoàn tác một commit và tìm hiểu cách khôi phục mã nguồn khi có lỗi.  
4.Dự án đáp ứng đầy đủ yêu cầu: có hơn 10 commit, tối thiểu 3 branch, 2 pull request, thực hành conflict và revert. Qua dự án, em hiểu rõ hơn về quy trình quản lý mã nguồn bằng Git và GitHub.  

**các lệnh đã được sử dụng trong project là:**  

git clone <repository_url>: Tải repository từ GitHub về máy tính.  
git config --global user.email "email@example.com": Thiết lập email cho Git.  
git status: Kiểm tra trạng thái các file trong dự án.  
git branch: Hiển thị danh sách các branch hiện có.  
git checkout <branch>: Chuyển sang branch được chỉ định.  
git checkout -b <branch>: Tạo branch mới và chuyển sang branch đó.  
git add .: Đưa tất cả thay đổi vào vùng staging.  
git commit -m "Message": Tạo commit và lưu lại thay đổi.  
git push origin <branch>: Đẩy code từ máy lên GitHub.  
git pull origin <branch>: Cập nhật code mới nhất từ GitHub về máy.  
git merge <branch>: Hợp nhất branch được chỉ định vào branch hiện tại.  
git log --oneline --all --graph: Xem lịch sử commit dưới dạng sơ đồ.  
git revert HEAD: Hoàn tác commit gần nhất bằng một commit mới.  
git merge main: Thực hành gộp code từ branch main.
git add . && git commit -m "Resolve merge conflict": Lưu kết quả sau khi xử lý xung đột.
Pull Request (trên GitHub): Đề xuất hợp nhất code từ branch tính năng vào branch chính.
Merge Pull Request (trên GitHub): Hợp nhất Pull Request vào branch main.
Resolve Conflict (trên GitHub/Git): Giải quyết xung đột khi hợp nhất code từ nhiều branch.

 **Em sẽ dùng Git như thế nào khi Vibe Code với AI**

Khi làm việc với AI để hỗ trợ lập trình (Vibe Coding), em sẽ sử dụng Git để quản lý và kiểm soát các thay đổi trong mã nguồn một cách hiệu quả. Trước khi thử một ý tưởng hoặc đoạn mã do AI đề xuất, em sẽ tạo branch riêng để phát triển và kiểm thử. Trong quá trình làm việc, em sẽ commit thường xuyên với nội dung rõ ràng để dễ dàng theo dõi lịch sử thay đổi và quay lại phiên bản ổn định khi cần. Khi hoàn thành một tính năng, em sẽ sử dụng Pull Request để xem xét lại mã nguồn trước khi hợp nhất vào branch chính. Nếu AI tạo ra lỗi hoặc thay đổi không mong muốn, em có thể sử dụng Git để so sánh, khôi phục hoặc revert các commit. Nhờ đó, Git giúp em tận dụng sức mạnh của AI nhưng vẫn đảm bảo mã nguồn được quản lý chặt chẽ, an toàn và dễ bảo trì.
